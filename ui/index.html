Supply Chain Flow
🏭
Supplier
localhost:3001
Holds raw material secrets. Generates Proof₁ that water & recycled-content meet the threshold — without revealing actual values.
→
🔩
Manufacturer
localhost:3002
Receives Proof₁, combines with its own assembly secrets, and generates Proof₂ (the outer ZKP) for the EU Verifier.
→
🏛️
EU Verifier
this browser
Calls /verify and checks the returned Proof₂. Accepts or rejects based purely on the cryptographic proof — no raw data needed.

Run Demo Scenario
batch_id=0x42
Happy path — clean supplier data and good assembly efficiency produce a valid Proof₂.
batch_id=0x99
Supplier's water usage was too high; no Proof₁ was ever issued, so the manufacturer has nothing to build on.
force_swap=true
Manufacturer tries to reuse a proof from a different batch. The circuit binding detects the mismatch and rejects it.
batch_id=0xCD
Supplier's proof is fine but the manufacturer's own assembly efficiency falls below the threshold, causing Proof₂ to fail.
force_forge=true
Server tampers with the proof bytes before returning them. The browser detects invalid hex and flags the proof as forged.
